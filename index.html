Welcome to the Magic of Firebase

This is meant to be a basic introduction to Firebase.

The main goals are to introduce the role and value of the 'backend' when building web applications and begin to get you familiar with Firebase as a backend technology.

Enter the Backend

Lets take a look at some applications we are familiar with.

Operation Peach is a game developed by a team in one of the Mission Bit 2014 spring semester Intro Web Programming classes. Open up Operation Peach in two different browsers sized to view side by side on your computer. Begin to play both games. Describe what the application is doing. Is this the behavior you'd expect? Now refresh the browsers.

In a new tab in each browser navigate to Amazon . Log into your Amazon account in both browsers. In browser 'alpha' go to your primary 'wish list' or create a new wish list if you don't have one. In browser 'beta' add three items to your wish list. Pay attention to what happens in 'alpha' each time you add an item to your wish list in 'beta'. Now refresh 'alpha' and take note of behavior.

In a new tab in each browser navigate to Twitter . Log into your Twitter account in both browsers. In both browsers go to your profile page (twitter.com/username). In 'alpha' send a tweet. Observe 'beta' for 90 seconds. Any action? In 'beta' scroll down the page. Any action? Now refresh both browsers.

How would you describe what the backend is doing in the Amazon and Twitter applications?

The backend for Amazon and Twitter are recording and storing data and the users interactions with the data.

Extra: How would your categorize key differences in the Amazon and Twitter data sets?

Time to Build

Build a simple help request system with the following functionality

User Page. On this page of the app, the user submits a message about what they need help with.

  • Include two input fields. One for the user name and one for the help message
  • Include a running list, ordered chronologically, of the user names in the queue to be helped. The message does not need to be displayed in the user queue.

Helper (customer service representative) Page. On this page of the app, the helper is able review the requests and intitiate help.

  • Display a running list, ordered chronologically, of the user names and their help messages.
  • With each help message include a 'help' button.
  • When the 'help' clicks the help button. Two things happen. The request is removed from the 'helper' queue and the username is removed from the user view of the queue. Additionally, the user and the helper are taken to a new page 'help chat' page where the helper can asked additional questions from and provide answers to the user. Extra: on this 'help chat' page have the first message displayed be the original 'help message' posted by the user
